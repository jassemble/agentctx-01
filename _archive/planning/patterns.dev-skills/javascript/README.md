# JavaScript Patterns

Design patterns, performance patterns, and rendering patterns for JavaScript and TypeScript. From [patterns.dev/vanilla](https://www.patterns.dev/vanilla/).

| Skill | Type | Description |
|-------|------|-------------|
| [`singleton-pattern`](singleton-pattern/SKILL.md) | Design | Share a single global instance throughout our application |
| [`observer-pattern`](observer-pattern/SKILL.md) | Design | Use observables to notify subscribers when an event occurs |
| [`proxy-pattern`](proxy-pattern/SKILL.md) | Design | Intercept and control interactions to target objects |
| [`prototype-pattern`](prototype-pattern/SKILL.md) | Design | Share properties among many objects of the same type |
| [`module-pattern`](module-pattern/SKILL.md) | Design | Split up your code into smaller, reusable pieces |
| [`mixin-pattern`](mixin-pattern/SKILL.md) | Design | Add functionality to objects or classes without inheritance |
| [`mediator-pattern`](mediator-pattern/SKILL.md) | Design | Use a central mediator object to handle communication between components |
| [`flyweight-pattern`](flyweight-pattern/SKILL.md) | Design | Reuse existing instances when working with identical objects |
| [`factory-pattern`](factory-pattern/SKILL.md) | Design | Use a factory function in order to create objects |
| [`command-pattern`](command-pattern/SKILL.md) | Design | Decouple methods that execute tasks by sending commands to a commander |
| [`provider-pattern`](provider-pattern/SKILL.md) | Design | Make data available to multiple child components |
| [`loading-sequence`](loading-sequence/SKILL.md) | Performance | Optimize your loading sequence to improve how quickly your app is usable |
| [`static-import`](static-import/SKILL.md) | Performance | Import code that has been exported by another module |
| [`dynamic-import`](dynamic-import/SKILL.md) | Performance | Import parts of your code on demand |
| [`import-on-visibility`](import-on-visibility/SKILL.md) | Performance | Load non-critical components when they are visible in the viewport |
| [`import-on-interaction`](import-on-interaction/SKILL.md) | Performance | Load non-critical resources when a user interacts with UI requiring it |
| [`route-based`](route-based/SKILL.md) | Performance | Dynamically load components based on the current route |
| [`bundle-splitting`](bundle-splitting/SKILL.md) | Performance | Split your code into small, reusable pieces |
| [`prpl`](prpl/SKILL.md) | Performance | Optimize initial load through precaching, lazy loading, and minimizing roundtrips |
| [`tree-shaking`](tree-shaking/SKILL.md) | Performance | Reduce the bundle size by eliminating dead code |
| [`preload`](preload/SKILL.md) | Performance | Inform the browser of critical resources before they are discovered |
| [`prefetch`](prefetch/SKILL.md) | Performance | Fetch and cache resources that may be requested some time soon |
| [`third-party`](third-party/SKILL.md) | Performance | Reduce the performance impact third-party scripts have on your site |
| [`virtual-lists`](virtual-lists/SKILL.md) | Performance | Optimize list performance with list virtualization |
| [`compression`](compression/SKILL.md) | Performance | Reduce the time needed to transfer scripts over the network |
| [`islands-architecture`](islands-architecture/SKILL.md) | Rendering | Small, focused chunks of interactivity within server-rendered web pages |
| [`view-transitions`](view-transitions/SKILL.md) | Rendering | Animate page transitions using the View Transitions API |
